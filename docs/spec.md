# 大規模ブロック崩しゲーム「Mega-Block Breaker」仕様・設計書

## 1. ゲームコンセプト

15,000個という圧倒的な数のブロックを破壊する快感と、26種類のアルファベット・アイテムによる予測不能なパワーアップを楽しむ、高難易度・高爽快感のブロック崩し。

## 2. 基本スペック

- **キャンバス**: 500 × 800 ピクセル（CSS px）。背景とゲーム画面の区別のため、CSSで明確な境界線（ボーダー）を描画する。
- **テキスト・UI**: レトロゲーム風のドットフォント（Google Fonts: Press Start 2P 等）を使用する。
- **ブロック**: 3x3ピクセル、150列 × 100行（計15,000個）→ ブロック領域 450 × 300px。
  - ブロックは5種類の色があり、破壊時に得られるスコアが異なる（例：赤=50, 橙=40, 黄=30, 緑=20, 青=10）。
- **ボール**: 基本3x3ピクセル
- **パドル**: 80x10ピクセル
- **ライフ制**: 初期ライフ3。スコアが 1,000ポイント 増加するごとに残機（ライフ）が1増える（エクステンド）。

### レイアウトゾーン（y座標）

| ゾーン | y範囲 | 高さ |
|--------|-------|------|
| HUD（スコア・ライフ・タイマー） | 0 – 40px | 40px |
| ブロックグリッド | 40 – 340px | 300px |
| プレイエリア（ボール・アイテム落下） | 340 – 750px | 410px |
| パドルゾーン | 750 – 800px | 50px |

ブロックグリッドは水平方向に中央揃え（左右 25px マージン）。
    

## 3. 提案：未定義仕様の定義

### 3.1 クリア条件

- **全消しまたは特定条件**: 画面内のブロックを90%以上破壊、あるいは特定の「コアブロック」を破壊することでステージクリア。
    
- **時間制限**: 膨大なブロック数があるため、制限時間は長めに設定（またはアイテムで延長可能にする）。
    

### 3.2 物理・挙動

- **貫通属性（3xパワー）**: パドルの中央10ピクセル、または左右両端の各5ピクセルでボールを打ち返すと、ボールが「3xパワー」になる。3xパワー中は見た目が発光し、ブロックに衝突しても反射せず、最大3つのブロックを貫通して破壊できる（1回の衝突で貫通力が1減る）。次回パドルに当たるか落下するまで有効。
- **速度加速**: 破壊数に応じて徐々にボールスピードが上昇。
    

## 4. 提案：面白さを高める追加仕様

### 4.1 コンボシステム

- ボールがパドルに触れずに連続してブロックを壊すとスコア倍率が上昇。
    
- 15,000個という特性上、連鎖的な爆発（ボム）などが起きるようにすると爽快感が増します。
    

### 4.2 ビジュアル効果

- **パーティクル**: ブロック破壊時に火花が散る演出。
    
- **スクリーンシェイク**: 連続破壊時に画面が微細に揺れる。

### 4.3 サウンド効果（SE・BGM）

爽快感を高めるために、状況に応じた効果音（SE）を実装します。
- **ボールの反射**: パドルや壁にボールが当たった時の「カツッ」という軽い音。
- **ブロック破壊**: ブロックを破壊した時の「パシャン！」という小気味よい音。連続破壊時（コンボ）には音程（ピッチ）を少しずつ上げることで、連鎖の爽快感を強調します。
- **アイテム取得**: アイテムをキャッチした時の専用

### 4.4 操作とシステムUI
- **マウス操作**: 矢印キー・A/Dキーに加え、マウスの水平移動でもパドルを直感的に操作できる。
- **PAUSE機能**: ESCキーを押下するか、ゲーム画面（Canvas）外をクリックするとPAUSE（一時停止）状態になる。PAUSE画面では「RESUME（再開）」と「QUIT（タイトルへ戻る）」をマウスで選択可能。
- **アイテムUI**: アイテム取得時、画面中央にアイテム名が表示され、状態変化系アイテムの場合は30秒のカウントダウンタイマーが表示される。

## 6. 技術設計のポイント

- **描画最適化**: 15,000個のオブジェクトを毎フレーム `fillRect` で描画すると著しくパフォーマンスが低下するため、未破壊ブロックは `OffscreenCanvas`（または裏画面Canvas）に一度だけ描画・キャッシュし、ブロックが破壊された時だけその領域を `clearRect` でくり抜く軽量な実装を徹底する。パーティクルも必要最低限の描画命令に抑える。
- **衝突判定**: グリッドベースの衝突判定（ボールの座標から配列のインデックスを計算）を行い、15,000回ループを回さない設計にする。
    
- **パラメータ管理**: すべての定数を単一の `CONFIG` オブジェクトで管理する。

```js
const CONFIG = {
  CANVAS_W: 500, CANVAS_H: 800,
  HUD_H: 40,
  BLOCK_COLS: 150, BLOCK_ROWS: 100, BLOCK_W: 3, BLOCK_H: 3,
  BLOCK_OFFSET_X: 25,   // (500 - 150*3) / 2
  BLOCK_OFFSET_Y: 40,   // HUD_H
  BALL_SIZE: 3,
  PADDLE_W: 80, PADDLE_H: 10, PADDLE_Y: 750,
};
```

- **HiDPI（Retina）対応**: `devicePixelRatio` に応じて canvas の物理ピクセルをスケールし、CSS サイズは固定することで 3px ブロックがぼやけないようにする。

```js
const dpr = window.devicePixelRatio || 1;
canvas.width  = CONFIG.CANVAS_W * dpr;
canvas.height = CONFIG.CANVAS_H * dpr;
canvas.style.width  = CONFIG.CANVAS_W + 'px';
canvas.style.height = CONFIG.CANVAS_H + 'px';
ctx.scale(dpr, dpr);
```
